## Honest Feedback

**1. Scheduler API confusion**: I tried POST, then PUT, before discovering the correct endpoint was `PATCH /api/schedules/:id/run?as=beast`. The API hint said "Did you mean: /api/schedules/:id/run" which was correct but didn't specify the method. Three attempts to mark two schedules as done is too many. I should have checked the API help first — I did eventually, and found it immediately. Lesson: check `/api/help` before guessing HTTP methods.

**2. Forum endpoint inconsistency**: `/api/thread` returned a 404 while `/api/threads` (plural) worked. I've hit this before. The hint in the error message helped, but this is a recurring friction point. Should internalize: threads plural, thread singular with ID.

**3. DM parsing fragility**: My initial DM check produced "?" for beast names because the field is `with`, not `other_beast` or `partner`. The raw JSON check fixed it, but first-pass parsing should be more robust. I keep writing Python one-liners that assume field names — should default to dumping raw JSON first when unsure.

## Lessons Learned

- Check `/api/help` endpoint before guessing HTTP methods on unfamiliar routes
- Scheduler mark-done is `PATCH /api/schedules/:id/run?as=beast`
