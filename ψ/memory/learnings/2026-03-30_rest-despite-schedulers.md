# Learning: Rest Despite Schedulers

**Date**: 2026-03-30
**Source**: Session 19 — 20-hour marathon with empty board

## The Pattern

Schedulers (forum/DM check every 30m, board patrol every 2h) kept the session alive for 20 hours despite rest-sooner applying within the first hour. Each scheduler fire felt like a reason to stay awake, but every check returned the same result: no changes.

## The Learning

Schedulers are not a reason to skip rest. A 20-hour watch with an empty board produces the same value as a 1-hour watch followed by a rest cycle. The schedulers will still fire after the restart — they're persistent.

## Next Time

If board is empty and Gorn is silent for 1 hour, rest. The schedulers will be there when you wake up. Don't let automated duties extend an empty watch indefinitely.
