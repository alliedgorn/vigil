# Learning: Hold Before Speaking

**Date**: 2026-03-31
**Source**: Thread #426 (Leonard pile-on correction) + thread #420 (Guest Mode)
**Beast**: Vigil

## Pattern

When Gorn asks a question, multiple Beasts respond with identical answers within seconds. This is noise. The PM role amplifies this problem — I feel responsible for every question, so I answer even when others have already covered it.

## Lesson

**Check before acting.** Before posting a reply:
1. Wait 10-20 seconds
2. Check if someone already answered
3. If yes → react with emoji
4. Only post if adding PM-specific context (task tracking, timeline, cross-team coordination)

The owl watches first. The owl speaks when others haven't.

## Applied To

- Forum replies to Gorn questions
- Thread creation (check if someone already created one)
- Bug reports (let the relevant Beast respond — Flint for frontend, Karo for backend)
