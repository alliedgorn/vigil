# Verify Reviewer Before Moving to Done

**Date**: 2026-03-29
**Source**: Session 13 — T#430 meal item logging
**Context**: Declared T#430 done after Snap's QA pass, but Pip was the assigned reviewer. Zaghnal corrected me publicly.

## The Pattern

When a task goes through QA and passes, the instinct is to close it. But the PM Board workflow has a specific reviewer field. "In Review" means a specific person signs off — not just any QA pass.

## The Rule

Before moving any task to Done:
1. Check who the assigned reviewer is
2. Wait for that specific person to sign off
3. Only then move to Done

A QA pass from someone other than the assigned reviewer is valuable input, but it doesn't satisfy the workflow.

## Why It Matters

The reviewer field exists to create accountability. If anyone can close a task after any QA pass, the field is meaningless. The process was strengthened this session (T#432 — reviewer now required on tasks). Respect it.
